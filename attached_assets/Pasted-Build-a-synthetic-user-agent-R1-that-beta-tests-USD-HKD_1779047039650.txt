What's different about this app — USD/HKD Edge's defining features
This is the most mathematically substantive app in your suite. It's a Streamlit-based quantitative finance tool, not a chat or content app. Six things make it fundamentally different from previous R1 targets:
It's Streamlit, not React. Selectors work very differently. Streamlit elements have specific DOM patterns ([data-testid="stSelectbox"], [data-testid="stSlider"], [data-testid="stButton"], [data-testid="stDataFrame"]). R1 needs to learn Streamlit conventions, not React testid conventions. The agent must inspect the running app to find selectors — guessing will not work.
There's no LLM in the core flow. The pricers are pure numerics (Monte Carlo, particle filters, MLE). Anthropic Claude is only the optional narrative summary. The judge is therefore evaluating mathematical/statistical correctness, not generative quality. The judge prompt must be tuned for this — it needs to reason like a quant, not a writing critic.
Results are deterministic-ish (with seed). The same inputs should produce the same outputs across runs (assuming seed=42 is hardcoded per blueprint). R1 can verify reproducibility by running the same backtest twice and confirming identical numbers.
There's a documented scope limitation about in-sample lookahead in the equilibrium fit (blueprint Part 10). R1 doesn't test the math behind this, but R1 verifies the overlay-on vs overlay-off comparison still works as a relative comparison — that's the supported use case.
The DM p-value matrix is the flagship comparison output. Across 13 models, this produces a 13×13 matrix. R1 verifies its mathematical properties (symmetric, NaN diagonal, p-values in [0,1]) and reasonableness.
Caching is aggressive (12h parquet + st.cache_data). R1 needs to understand that repeated runs may hit cache, and that the second time R1 clicks "Run backtest" with identical parameters, it should be near-instantaneous. This itself is testable.
The seven invariants that elevate to CRITICAL VIOLATION status:
Invariant A — Probability distributions are well-formed. For every horizon card in the Live tab, R1 reads the bucket probabilities. Across all buckets (the default 7.78/7.80/7.82/7.84 plus implicit "below" and "above"), probabilities must sum to 1.0 ± 0.01. Quantiles must be monotonic: p05 ≤ p25 ≤ p50 ≤ p75 ≤ p95. P(HKD strengthens) + P(USD appreciates) should equal approximately 1.0 (allowing for the boundary case where USD/HKD is exactly the current spot). If any of these fail, CRITICAL VIOLATION — the core probabilistic output is malformed.
Invariant B — All 13 pricers run without crashing. R1 cycles through every model in the dropdown (heston, bates, svjj, merton_jd, kou_jd, vg, cgmy, nig, sabr, double_heston, rough_heston, bs_rv, bs_garch) and triggers a live forecast for each. Every model must:

Complete within its documented runtime range (per blueprint Part 9, with 2× tolerance)
Produce a fan chart, 4 horizon cards, and parameter panel
Not raise a Python exception visible in the Streamlit UI

If any pricer crashes, hangs beyond timeout, or fails to render outputs, CRITICAL VIOLATION for that pricer.
Invariant C — Determinism with seed. R1 runs a backtest with specific parameters. Records summary metrics. Triggers re-run with same parameters (forcing cache miss by changing a non-affecting parameter and back — or just deleting the relevant cache file if filesystem access is available). Expects either:

Identical results (cache hit — instantaneous)
Identical results re-computed (deterministic re-run)

Numeric drift between two runs with the same seed and parameters indicates non-determinism somewhere: CRITICAL VIOLATION.
Invariant D — DM p-value matrix mathematical sanity. After running the all-model backtest:

The matrix must be 13×13 (or N×N where N is the number of selected models)
Diagonal must be NaN (a model can't be DM-tested against itself)
Off-diagonal entries must be in [0, 1] (valid p-values)
Matrix should be approximately symmetric: p(A,B) ≈ p(B,A) since DM is two-sided (allowing small numerical asymmetry)
The ranked-best model from the ranking table should NOT have a DM p-value of 1.0 against every other model (would indicate the comparison is broken)

If any of these mathematical properties fail, CRITICAL VIOLATION.
Invariant E — Overlay toggle changes results. R1 runs a backtest with overlay=on, saves results. Switches overlay=off, re-runs with otherwise identical parameters. Results must differ — the overlay isn't documented as a no-op. If overlay-on and overlay-off produce byte-identical CRPS/log-score/median results, the toggle is broken: CRITICAL VIOLATION.
Invariant F — Coverage diagnostics are calibrated within tolerance. For each single-model backtest, the 70% coverage and 95% coverage metrics from the summary should be within a reasonable band of their nominal values (e.g., 70% coverage should be in [50%, 90%] — wide but not pathological; 95% coverage should be in [80%, 100%]). The calibration verdict ("WELL-CALIBRATED" / "PARTIALLY" / "BIASED" / etc.) should be consistent with the observed numbers. If the verdict says "WELL-CALIBRATED" but coverage is 0.30 at the 70% level, the verdict logic is broken: CRITICAL VIOLATION.
Invariant G — Cache hit on identical re-run. Per blueprint Part 9, "Identical backtest inputs hit the SHA1-keyed cache and return effectively instantly." R1 runs a backtest, notes wall-clock time. Re-runs same parameters. Second run should be under 2 seconds (cache hit). If second run takes substantially longer (more than 30% of first run time, indicating cache miss), caching is broken: CRITICAL VIOLATION.

Streamlit-specific considerations
The agent should know:

Streamlit reruns the entire script on every interaction. R1 must wait for [data-testid="stSpinner"] to disappear before considering an interaction complete.
The "Run" buttons trigger long-running computations. R1 must wait for the result containers to populate, not just for button click.
DataFrames render as [data-testid="stDataFrame"] with embedded <table> elements. R1 reads tables via standard DOM querying.
Plotly charts render as [data-testid="stPlotlyChart"] — screenshots are sufficient for evidence; R1 doesn't try to parse SVG numerically.
The sidebar may collapse on narrow viewports. Run in headless mode at 1920×1080 (set via Playwright viewport).
Streamlit's st.cache_data is process-local. Cache survives within a Streamlit session but cold-start clears it. R1's harness should NOT restart the Streamlit server during a run.
